### What Is owvision?

**open weather vision** (in short _owvision_) is an open source software system, that allows you to build your own weather control center.

Currently it has four main features.

<Card title="Bare Metal Features">
    <ol>
        <li>⏲️ View your station's most recent sensor readings in realtime</li>
        <li>〽️ Navigate through your recorded historic weather data*</li>
        <li>🏆 Checkout historic highs and lows*</li>
        <li>
            ☁️ View a 7 day forecast for your weather station's set up location (via
            [Open-Meteo](https://open-meteo.com/))*
        </li>
    </ol>
    <i style="font-size:0.8em">*Still in development.</i>
</Card>

And all this for free, in an easy to use modern user interface. That's the idea behind owvision. Cool, isn't it😎?

2.  **Download the daemon (_owvi_)**

    _owvision_ is a software system, that consists of multiple components. The core component is the **daemon**. In short it manages the weather database and therefore should be always running. Don't install this component on your personal computer, instead prefer a Rapberry Pi or a (virtual) Linux server.
    Added to that this host should be visible in your network (or better the whole internet).

    **OS-Support**: Debian based linux (Ubuntu, Raspberry Pi OS, ...), Windows 10/11

3.  **Start the daemon**

    After downloading the installer, simply run it to initialize the daemon. The installer will automatically download the correct binary, setup the background service, add it to your system's PATH, and walk you through the configuration process. If you have any issues here, don't hesitate to ask for help on our [discord server]().
    The installer will prompt for admin privileges on windows if required.

    **On Linux:**

    ```sh
    chmod +x owvi_installer_*
    ./owvi_installer_*
    ```

    **On Windows:** Simply double click the downloaded `owvi_installer_winx64.exe` or run it from PowerShell.

    <i style="font-size: 0.8rem">\*The `.owvision` folder will be created automatically.</i>

4.  **Download the recorder (_owrec_)**

    The second very important component is the **recorder**. It is basically a simple buffer that is responsible for reading the station's current sensor readings using an interface and forwarding
    this information to the daemon.

    Install this component on the device that is directly connected to your weather station. This may be your server, but this is not mandatory.
    The recorder pushes data to the daemon, so the daemon has to be visible to the recorder.

    **OS-Support**: Debian based linux (Ubuntu, Raspberry Pi OS, ...), Windows 10/11

5.  **Start the recorder**

    After downloading the installer, simply run it to initialize the recorder. Similar to the daemon, this will download the core binary, create the service, and walk you through the setup. If you have any issues here, don't hesitate to ask for help on our [discord server]().
    The installer will prompt for admin privileges on windows if required.

    **On Linux:**

    ```sh
    chmod +x owrec_installer_*
    ./owrec_installer_*
    ```

    **On Windows:** Simply double click the downloaded `owrec_installer_winx64.exe` or run it from PowerShell.

    <i style="font-size: 0.8rem">\*The `.owvision` folder will be created automatically.</i>

6.  **Find, download and start your interface**

    Interfaces are community written. That's why their installation procedure is dependent on the particular station. Find your interface [here](/owvision/reference/supported-stations).

    To check if the recorder detected your weather station run:

    ```sh
    owrec status
    ```

    After some time the daemon should also detect your weather station, check it (on your server) using:

    ```sh
    owvi station list
    ```
